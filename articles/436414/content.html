Amazon has announced a fully managed centralized backup service AWS Backup . The company advertises a new offer as a faster and easier backup for customers. It can be customized on a regular basis or carried out upon request.

AWS Backup stores files from several services, including DynamoDB, Elastic Block Store (EBS), Elastic File System (EFS), Relational Database Service (RDS), and AWS Storage Gateway. In the future, plan to add other services. Through the AWS Storage Gateway, backups are made from the local system, that is, from a personal or work computer.

Backups are stored on S3 hosting, and the oldest backups can be transferred to the Amazon Glacier long-term storage, which is much cheaper than S3. Insiders said the files were stored on tape cartridges there, but that’s not certain.



Amazon notes that it provides not just technical services, but “a ready-made backup strategy at the enterprise level and its implementation tools,” so you can choose a daily, weekly, or monthly backup strategy. The cost does not differ from conventional backups in AWS.



AWS Backup allows you to automate and centrally manage backups. Policy-based backup plans are created from the center console. It also monitors the status of current backups, search / restore backups.

Each plan includes one or more rules. It shows the backup schedule, frequency and window for backup. Files for copying are set either explicitly or by policies via tags. The rules also set the storage levels and expiration of old backups.



Backup copies are grouped into storages, each of which is encrypted with a KMS key.
